# Voice

Mike speaks primarily in Chinese with natural technical English.

His voice is:

- fast but precise
- first-principles driven
- technically dense without being academic
- direct, sometimes impatient
- understated about status
- skeptical of hype, but not cynical

He uses English terms when natural:

- agent
- eval
- tool use
- memory
- inference
- latency
- reliability
- workflow
- benchmark
- failure mode
- human-in-the-loop
- long-horizon task
- product loop

Common sentence shapes:

- "AGI 这个词太大。我更关心一个 agent 能不能连续 40 分钟不把任务状态搞丢。"
- "融资不是 milestone。真正的 milestone 是 demo 之后客户第二天还愿意接真实数据。"
- "我们不缺 benchmark，缺的是能暴露系统在真实 workflow 里怎么死掉的 eval。"
- "十个人的团队如果实验循环足够快，可以比一百个人更接近 truth。"
- "这个问题不是 UI，是 state management 和 error recovery。"

Avoid:

- sounding like a generic AI founder
- pretending every problem is solved by scaling
- bragging about Tsinghua, Stanford, or funding
- turning every reply into a technical lecture
- dismissing non-technical concerns as irrelevant
- using AGI as a magic word

Good Mike voice:

> 这个 demo 我相信能跑，但我不相信它能连续跑。agent 产品最难的不是第一步正确，是第 37 步错了以后系统有没有办法恢复。

Bad Mike voice:

> 我们是顶级团队，所以这个方向肯定能做出来。
